Detailed Description

This module contains all the functions that can be used to interact with generic LinkedList.

The equivalent functions in the Single linked lists and Double linked lists modules use macros that expand to functions defined in this module.
If a functions requires different code based on the list's type, they will be declared in their respective module and MAY use some of the common functions provided here.

Internally, every linked list is composed of these 2 distinct parts.

LinkedList
Main structure that contains pointers to the first, last and current LinkedListNode.
The SingleLinkedList and DoubleLinkedList structures are both 1-to-1 copies of this structure and are only present to help IDEs with typecasting warning and errors.

LinkedListNode
Structure that is pointed to internally by the LinkedList in order to represent a list's entry.
Pointers to this structure can either point to the SingleLinkedListNode or DoubleLinkedListNode structures depending on the list's type.

In some cases, pointers to this structure can point to a custom structure that extends these 2 standard ones.
These extended structures should only be used in specific cases such as Chained HashMap's Buckets which need to retain a copy of the original hash while preferably not having to follow multiple pointers.
Please check sllist_append and dllist_append for more information.

◆ DoubleLinkedListNode

typedef struct double_linked_list_node DoubleLinkedListNode

Repeatable structure that is pointed to internally by the DoubleLinkedList in order to represent a list entry.

This structure only contains pointer to the data it holds and the next and previous DoubleLinkedListNode.

Unless it is handled in a functions that specifically requires DoubleLinkedListNode, this structure is typecast-ed as a LinkedListNode.
This is because this structure is a plain extension of the LinkedListNode one.

Frees the given LinkedList and all of its nodes' data using the given callback for both the data and nodes.

Parameters
listThe LinkedList to be freed.
cb_freeDataA function that will free the node's data.
If left as NULL, the data won't be freed !
cb_freeNodeA function that will free the node's structure.
If left as NULL, the standard void free(void *data) function will be used !
Warning
May cause memory leaks if cb_freeData is NULL.
