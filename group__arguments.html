Warning:
Any reference to a sub-verb actually refers to a regular Verb which can be considered as the another verb's children.

Any reference to np_args_char can be the equivalent of char or wchar_t depending on 2 compiler constants.
These constants, NP_WIN32 and NP_ARGS_WCHAR, if both used, will make all functions and structures in this module use wchar_t strings instead of char ones.
This features allows you to easily handle unicode characters and emojis in launch arguments on Windows platforms.

These macro-ed types and functions are defined in arguments/macros.h.
This header also defines a bunch of macros for string-related operations done in this module.
They shouldn't be used outside of it !

If you want an example of this feature on a Windows app, you should check aziascreations/C99-Win32-Wifi-Utility on GitHub.

Enumerator
ERROR_ARGUMENTS_NONE 

Indicates that the parser encountered no error.

ERROR_ARGUMENTS_INSERTION_FAILURE 

Indicates that the parser failed to insert a value in an option. (Likely due to a malloc failure)

ERROR_ARGUMENTS_UNKNOWN_OPTION 

Indicates that an unknown option was given as a launch argument.

ERROR_ARGUMENTS_NO_DEFAULT_FOUND 

Indicates that a default Option with the FLAG_OPTION_DEFAULT flag was needed but couldn't be found in the current Verb.
This error is likely caused by improperly formatted launch arguments.

ERROR_ARGUMENTS_DUAL_END_OF_OPTIONS 

Indicates that the -- token was encountered more than once.

ERROR_ARGUMENTS_SINGLE_OPTION_REUSED 

Indicates that an Option without the OPTION_FLAG_REPEATABLE flag was found more than once in the launch arguments.

ERROR_ARGUMENTS_NO_ARGUMENTS_LEFT 

Indicates that an Option that required a value couldn't be parsed since the parser reached the end of the parsable launch arguments.

ERROR_ARGUMENTS_REQUIRED_OPTION_UNUSED 

Indicates that an option with the FLAG_OPTION_REQUIRED flag wasn't given when ont of its parent Verb was used.

ERROR_ARGUMENTS_OPTION_HAS_VALUE_AND_MORE_SHORTS 

Indicates that an Option that requires a value was given as its token in a multi-token group but wasn't located at the end of said group.

ERROR_ARGUMENTS_THIS_SHOULD_NOT_TRIGGER 

Special error that should never be returned.
It is used in duplicated checks which should never fail if the one in which this is present is reached.

Enumerator
FLAG_OPTION_NONE 

Used by default to indicate that the Option has no special features.

FLAG_OPTION_DEFAULT 

Indicates that an Option should be a default one, meaning that it can be used without using the short and long argument.
Its usage will be automatically detected in the parser.

FLAG_OPTION_HAS_VALUE 

Indicates that an Option can have at least one value.

FLAG_OPTION_REPEATABLE 

Indicates that an Option can be used multiple time with or without a value.
The usage count can be found in arguments_option::occurrences.

FLAG_OPTION_HAS_MULTIPLE_VALUE 

Indicates that an Option can have one or more value(s).

FLAG_OPTION_HIDDEN 

Indicates that an Option shouldn't be shown in the help text.

FLAG_OPTION_REQUIRED 

Indicates that an Option must be used in a used verb Verb.

FLAG_OPTION_STOPS_PARSING 

Indicates that an Option must stop the parsing process as soon as it has been parsed.

FLAG_OPTION_ALLOW_VERBS_AFTER 

Indicates that an Option will not prevent further verbs from being parsed when encountered.
Usage of this flag required some forethought if the Option uses the FLAG_OPTION_DEFAULT flag since its values may be interpreted as a verb and cause all kinds of errors.
A "better" alternative would be to only use the FLAG_OPTION_REQUIRED flag if the value is needed, which will force the user to explicitly use the option's name or token.

◆ args_parseArguments()

enum EArgumentParserErrors args_parseArguments ( Verb *  rootVerb,
np_args_char *  arguments[],
int  startIndex,
int  endIndex,
Verb **  pRelevantVerb 
)

Parses launch arguments following a given Verb and all its members until the end is reached or an error occurs.
In the event of a failure, the entire rootVerb should be considered as tainted and not used in any subsequent task that doesn't involve it being freed from memory.

Parameters
rootVerbThe root verb from which the parsing process will start.
argumentsArray of strings to be parsed as the launch arguments.
startIndexThe index from which parsing will take place in the given arguments array.
endIndexThe index at which parsing will stop in the given arguments array.
pRelevantVerbA nullable pointer to a Verb pointer used as the return value which will point to the last used verb.
Returns
An error code indicating how the parsing process went and which error may have occurred.
